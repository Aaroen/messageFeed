# Agent Web Assistant Sliding Workspace Plan

## 1. 本轮目标

在 Web 顶部“订阅”“推荐”之后新增同一风格的“助理”入口，并实现与订阅、推荐一致的左右滑动切换。三个页面需要同时挂载和加载内容，而不是滑动到目标页后才加载。助理页复用现有 Agent 工作台和后端接入，但需要面向普通用户整理界面：最近任务不展示开发治理任务，评测基线不展示在用户界面，执行进度放到发起任务下方。

## 2. 当前依据

1. `/agent` 已由 `AgentPlanView.vue` 承载，并已接入 `createAgentTask`、任务列表、计划进度、审计事件和最终报告等后端内容。
2. 顶部“订阅”“推荐”页签由 `feedTabs` 配置和 `AppFeedHeaderContent.vue` 渲染。
3. 当前 `FeedPager` 已同时挂载订阅与推荐两个 pane，并通过 `useFeedPagerTransition` 支持左右滑动。
4. 当前 `FeedPager` 只支持双页索引，`/agent` 仍走普通页面路由，因此需要扩展为三页模型，不能只修改路由标题。
5. `AgentPlanView.vue` 目前混合展示了用户任务、工程治理摘要和评测基线，其中评测属于开发者验证内容，不应暴露给普通用户。

## 3. 实施范围

1. 在顶部页签配置中新增 `{ key: 'agent', label: '助理', path: '/agent' }`，位于“推荐”之后。
2. 将 `FeedPager` 扩展为三页：订阅、推荐、助理，并保持三个 pane 同时挂载。
3. 扩展 `useFeedPagerTransition` 的索引、目标路径、阻塞方向和程序化切换逻辑，使订阅、推荐、助理可以连续左右滑动。
4. 使 `/agent`、`/agent/plans/:id`、`/agent/plans/:id/evidence/:recordKey` 在顶部页签中高亮“助理”。
4. 将 `/agent` 路由标题从 `Agent 任务` 调整为 `助理`，与顶部入口一致。
5. 调整 `AgentPlanView.vue`：执行进度移动到发起任务下方；删除用户界面的评测基线展示和评测触发；最近任务只展示真实用户任务列表，不展示工程治理摘要。
6. 不新增后端接口，不新增冗余页面，不继续扩大开发者评测功能在用户界面的暴露面。

## 4. 非目标

- 本轮不实现新的 Agent 后端能力。
- 本轮不重构全部 `AgentPlanView.vue` 主体。
- 本轮不调整企业微信入口或最终报告投递语义。
- 本轮不删除后端评测 API 或服务能力；仅从用户界面移除入口和展示。
- 本轮不物理删除历史任务数据；仅确保普通用户工作台不展示开发治理任务。

## 5. 验收标准

1. 顶部主入口显示“订阅 / 推荐 / 助理”，且视觉风格一致。
2. 订阅、推荐、助理三页均在 `FeedPager` 中同时挂载，左右滑动可以连续切换。
3. `/agent` 及其计划详情页顶部均高亮“助理”。
4. 点击“助理”进入 `/agent`，页面展示现有 Agent 工作台和任务发起表单。
5. 执行进度显示在发起任务面板下方，不再位于页面最底部。
6. 最近任务面板不展示开发治理摘要；无真实用户任务时显示为空状态。
7. 评测基线不再出现在用户界面，也不在页面挂载时请求评测接口。
8. 订阅和推荐仍按原有信息流行为展示。
9. 前端类型检查、构建和测试通过。
