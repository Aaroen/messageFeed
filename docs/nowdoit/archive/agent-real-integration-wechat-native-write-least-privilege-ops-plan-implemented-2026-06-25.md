# Agent 真实联调企微原生按钮写能力最小权限与运维验收计划

**创建日期**：2026-06-25

## 1. 本轮目标

在 Agent 已具备 Web/企业微信任务入口、实时进度、任务工作台、部署演练、长期趋势、告警策略、企微动作 fallback、多用户压测摘要、企微回调联调摘要、写操作沙箱和端到端验收摘要的基础上，推进真实环境联调准备、企业微信原生按钮消息、写操作 capability 最小权限开放和上线前运维验收，使 Agent 进入更接近生产上线的可控交付状态。

## 2. 实施范围

1. 真实环境联调准备：
   - 输出真实 Web、企业微信回调、数据库迁移、worker、通知和 eval 的联调核验结构。
   - 保留联调状态、风险、阻断项和下一步动作。
2. 企业微信原生按钮消息：
   - 在现有文本 fallback 基础上，建立企业微信按钮消息的数据结构。
   - 覆盖查看进度、审批、重试、恢复、取消和查看最终报告动作。
   - 保持不支持原生按钮时的文本 fallback。
3. 写操作 capability 最小权限开放：
   - 选择最小范围写操作 capability 建立审批沙箱放行结构。
   - 默认仍保持拒绝或需审批。
   - 权限、预算、审批、执行和审计必须形成闭环。
4. 上线前运维验收：
   - 输出上线前检查摘要，覆盖配置、迁移、健康检查、worker、配额、告警、趋势、审计、回滚和人工接管。
   - 支持 Web 任务工作台查看验收状态。

## 3. 非目标

- 本轮不直接执行真实生产发布。
- 本轮不开放任意写操作 capability。
- 本轮不移除文本 fallback。
- 本轮不引入外部运维平台。

## 4. 验收标准

- Web 或 service 层可以查看真实环境联调准备摘要。
- 企业微信动作具备原生按钮数据结构和文本 fallback。
- 写操作 capability 最小权限策略默认受审批与审计约束。
- 上线前运维验收摘要覆盖配置、迁移、健康、worker、配额、告警、趋势、审计、回滚和接管。
- `go test ./...`、`go vet ./...`、`npm --prefix web run test`、`npm --prefix web run type-check`、`npm --prefix web run build` 通过。

## 5. 本轮实施清单

1. 后端任务工作台响应补充真实环境联调准备摘要、企业微信原生按钮动作摘要、写操作最小权限策略摘要和上线前运维验收摘要。
2. 真实环境联调准备摘要覆盖 Web、企业微信回调、数据库迁移、worker、通知、eval、风险、阻断项和下一步动作。
3. 企业微信原生按钮动作摘要基于现有 fallback 动作生成按钮数据结构，覆盖查看进度、审批、重试、恢复、取消和最终报告。
4. 写操作最小权限策略摘要默认保持拒绝或需审批，并输出允许候选、拒绝范围、审批、预算、权限和审计检查项。
5. 上线前运维验收摘要覆盖配置、迁移、健康、worker、配额、告警、趋势、审计、回滚和人工接管。
6. Web API 类型和任务工作台展示新增摘要，补充测试并执行完整验证。

## 6. 实施结果

1. 后端任务工作台响应已补充 `real_integration`、`wechat_native`、`write_least_privilege`、`ops_acceptance` 四类结构化摘要。
2. 真实环境联调准备摘要已覆盖 Web 入口、企微回调、数据库迁移、worker、通知、eval 和告警策略，并输出风险、阻断项和下一步动作。
3. 企业微信动作已从文本 fallback 扩展为原生按钮 schema，覆盖查看进度、审批、重试、恢复、取消和查看最终报告。
4. 写操作 capability 最小权限策略默认保持 `reject_or_require_approval`，允许候选限制为 `agent.schedule_message` 与 `agent.schedule_task`，拒绝高风险写入模式。
5. 上线前运维验收摘要已覆盖配置、迁移、健康、worker、配额、告警、趋势、审计、回滚、接管、压测、企微回调、写最小权限和生产演练。
6. 真实联调快照写入 `agent.real_integration_snapshot` audit，运维验收快照写入 `agent.ops_acceptance_snapshot` audit。
7. Web API 类型和任务工作台已展示真实联调、企微原生按钮、写最小权限和运维验收摘要。
