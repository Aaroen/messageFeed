# Agent 运营面板监控自动汇报写分阶段放量与反馈闭环计划

**创建日期**：2026-06-25

## 1. 本轮目标

在 Agent 已具备 Web/企业微信任务入口、实时进度、企业微信按钮直接控制、运行参数、监控回读、写放量建议、企业微信用户反馈和运行运营闭环摘要的基础上，推进可配置运营面板、监控异常自动汇报、写能力分阶段放量和企业微信用户侧闭环运营，使 Agent 运行状态具备更明确的运营配置、异常汇报和反馈处理路径。

## 2. 实施范围

1. 可配置运营面板：
   - 将运行参数摘要推进为运营面板配置摘要。
   - 覆盖参数组、展示项、刷新频率、告警入口、写放量入口和企微反馈入口。
2. 监控异常自动汇报：
   - 将监控回读摘要推进为异常自动汇报摘要。
   - 覆盖异常检测、企业微信发送、Web 可见性、日报关联和审计证据。
3. 写能力分阶段放量：
   - 将写放量建议推进为阶段化放量摘要。
   - 覆盖当前阶段、下一阶段、进入条件、退出条件、回滚条件和默认拒绝策略。
4. 企业微信用户侧闭环运营：
   - 将用户反馈摘要推进为反馈处理摘要。
   - 覆盖完成反馈、失败反馈、按钮反馈、Web 追踪反馈、处理状态和用户下一步。
5. 运营闭环总览：
   - 汇总运营面板、监控自动汇报、分阶段放量和反馈处理状态。

## 3. 非目标

- 本轮不开放未进入白名单的写 capability。
- 本轮不移除企业微信文本 fallback。
- 本轮不执行不可回滚的真实生产变更。
- 本轮不绑定单一外部监控 SaaS 的私有 API。

## 4. 验收标准

- 运营面板配置摘要可在 Web 或 service 层查看，并写入审计事件。
- 监控异常自动汇报摘要覆盖企业微信发送、Web 可见性、日报关联和审计证据。
- 写能力分阶段放量摘要保持最小权限、审批、预算、审计和回滚约束。
- 企业微信反馈处理摘要覆盖完成、失败、按钮、Web 追踪、处理状态和下一步。
- 运营闭环总览纳入运营面板、监控自动汇报、分阶段放量和反馈处理状态。
- `go test ./...`、`go vet ./...`、`npm --prefix web run test`、`npm --prefix web run type-check`、`npm --prefix web run build` 通过。

## 6. 实施结果

1. 后端任务工作台已新增五类运营闭环摘要：
   - `ops_panel_config`：运营面板配置摘要，覆盖参数组、展示项、刷新频率、告警入口、写放量入口和企微反馈入口。
   - `monitor_auto_report`：监控异常自动汇报摘要，覆盖异常检测、企业微信发送、Web 可见性、日报关联和审计证据。
   - `write_ramp_stage`：写能力分阶段放量摘要，覆盖当前阶段、下一阶段、进入条件、退出条件、回滚条件和默认拒绝策略。
   - `wechat_feedback_loop`：企业微信反馈处理摘要，覆盖完成反馈、失败反馈、按钮反馈、Web 追踪反馈、处理状态和用户下一步。
   - `operations_closed_loop`：运营闭环总览摘要，覆盖运营面板、监控自动汇报、分阶段放量、反馈处理和审计状态。
2. 后端审计已新增五类快照事件：

3. 前端任务工作台已新增运营闭环展示：
   - API 类型定义已同步新增五类响应结构。
   - `AgentPlanView.vue` 已展示运营面板、监控自动汇报、写分阶段放量、企微反馈处理和运营闭环总览摘要及检查项。
