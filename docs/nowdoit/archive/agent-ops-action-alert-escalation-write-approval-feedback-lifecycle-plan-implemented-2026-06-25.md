# Agent 运营动作异常升级写阶段审批与反馈工单生命周期计划

**创建日期**：2026-06-25

## 1. 本轮目标

在 Agent 已具备 Web/企业微信任务入口、实时进度、企业微信按钮控制、运营面板配置、监控异常自动汇报、写能力分阶段放量、企业微信反馈处理、运营闭环总览以及运营处理摘要的基础上，推进运营面板真实交互动作、异常升级通知策略、写能力阶段推进审批和反馈工单生命周期，使运营处理从摘要可见推进到可追踪、可审计、可审批的动作闭环。

## 2. 实施范围

1. 运营面板真实交互动作：
   - 将运营交互面板摘要推进为可执行动作定义。
   - 覆盖动作键、动作标签、处理入口、权限约束、幂等键和审计事件。
2. 异常升级通知策略：
   - 将异常去重升级摘要推进为升级策略摘要。
   - 覆盖升级等级、通知通道、重复抑制、升级接收方、恢复通知和审计证据。
3. 写能力阶段推进审批：
   - 将写阶段推进记录推进为审批摘要。
   - 覆盖审批状态、审批来源、目标阶段、授权范围、回滚门槛和默认拒绝策略。
4. 反馈工单生命周期：
   - 将企业微信反馈工单推进为生命周期摘要。
   - 覆盖创建、分派、处理中、等待用户、已关闭和失败转人工状态。
5. 运营动作闭环总览：
   - 汇总运营动作、异常升级、写阶段审批和反馈工单生命周期状态。

## 2.1 本轮实施清单

1. 后端任务工作台新增运营动作闭环摘要：
   - 新增运营面板动作定义摘要，覆盖动作键、动作标签、处理入口、权限约束、幂等键和审计事件。
   - 新增异常升级通知策略摘要，覆盖升级等级、通知通道、重复抑制、升级接收方、恢复通知和审计证据。
   - 新增写阶段推进审批摘要，覆盖审批状态、审批来源、目标阶段、授权范围、回滚门槛和默认拒绝策略。
   - 新增反馈工单生命周期摘要，覆盖创建、分派、处理中、等待用户、已关闭和失败转人工状态。
   - 新增运营动作闭环总览摘要，汇总运营动作、异常升级、写阶段审批和反馈工单生命周期状态。
2. 后端审计闭环：
   - 为上述五类摘要写入独立审计事件。

3. 前端任务工作台：
   - 扩展 API 类型定义。
   - 在任务工作台展示新增运营动作闭环摘要和关键检查项。
4. 验证与归档：
   - 执行 `go test ./...`、`go vet ./...`、`npm --prefix web run test`、`npm --prefix web run type-check`、`npm --prefix web run build`。
   - 写入实施结果和验证记录。
   - 归档当前文档并创建下一轮唯一活动文档。

## 3. 非目标

- 本轮不开放未进入白名单的写 capability。
- 本轮不绕过审批、预算、权限、审计和回滚约束。
- 本轮不执行不可回滚的真实生产变更。
- 本轮不移除企业微信文本 fallback。

## 4. 验收标准

- 运营动作定义可在 service 层和 Web 任务工作台查看，并写入审计事件。
- 异常升级通知策略覆盖通道、接收方、去重抑制、恢复通知和审计证据。
- 写阶段审批摘要保持最小权限、审批、预算、审计和回滚约束。
- 反馈工单生命周期覆盖主要状态和失败转人工路径。
- 运营动作闭环总览纳入运营动作、异常升级、写阶段审批和反馈工单生命周期状态。
- `go test ./...`、`go vet ./...`、`npm --prefix web run test`、`npm --prefix web run type-check`、`npm --prefix web run build` 通过。

## 5. 后续衔接

本轮完成后，继续推进运营动作真实 API 执行、升级通知投递回执、写阶段审批按钮化和反馈工单处理 SLA。

## 6. 实施结果

1. 后端任务工作台新增五类运营动作闭环摘要：
   - `ops_action_definition`：覆盖动作键、动作标签、处理入口、权限约束、幂等键和审计事件。
   - `alert_escalation_policy`：覆盖升级等级、通知通道、重复抑制、升级接收方、恢复通知和审计证据。
   - `write_stage_approval`：覆盖审批状态、审批来源、目标阶段、授权范围、回滚门槛和默认拒绝策略。
   - `feedback_ticket_lifecycle`：覆盖创建、分派、处理中、等待用户、已关闭和失败转人工状态。
   - `operations_action_closure`：汇总运营动作、异常升级、写阶段审批、反馈工单生命周期和审计状态。
2. 后端审计闭环已补齐：
   - 新增 `agent.ops_action_definition_snapshot`。
   - 新增 `agent.alert_escalation_policy_snapshot`。

   - 新增 `agent.feedback_ticket_lifecycle_snapshot`。
   - 新增 `agent.operations_action_closure_snapshot`。
3. 前端任务工作台已接入上述五类摘要：

   - 运营动作定义展示最多 6 项动作，所有新增摘要展示最多 8 项检查项。
4. 服务层测试已扩展：
   - 断言新增五类摘要关键字段有效。
   - 断言新增五类审计事件已写入。
