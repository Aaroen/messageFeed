# Agent 生产演练告警策略与企微组件计划

**创建日期**：2026-06-25

## 1. 本轮目标

在 Agent 已具备 Web/企业微信任务入口、实时进度、任务工作台、部署验收摘要、运行告警摘要和成本趋势摘要的基础上，推进生产部署演练、长期趋势留存、告警策略配置和企业微信原生交互组件适配，使 Agent 从可观测闭环进一步进入可演练、可配置和可操作的上线准备阶段。

## 2. 实施范围

1. 生产部署演练：
   - 增加可由 service 或管理接口触发的 Agent 部署演练记录。
   - 覆盖迁移、healthz、worker、Web 任务入口、企微任务入口、配额准入、通知审计和 eval 基线。
   - 演练结果写入结构化 metadata 或持久化记录，供 Web 查看。
2. 长期趋势留存：
   - 将成本、失败、通知、恢复和接管趋势从即时计算推进到可持久化留存。
   - 保留近期聚合读取能力，避免 Web 工作台依赖单次列表结果推断长期趋势。
3. 告警策略配置：
   - 为任务失败、通知失败、配额超限、低质量评分和长时间接管待处理增加可配置策略。
   - 支持 Web 设置页读取和更新 Agent 告警偏好。
   - 告警策略决策写入 audit，便于回溯。
4. 企业微信原生交互组件：
   - 在现有文本通知基础上，为审批、查看进度、重试、恢复和取消动作建立组件化消息结构。
   - 保持文本 fallback，避免不支持组件的通道丢失关键信息。

## 3. 非目标

- 本轮不接入外部 PagerDuty、飞书或短信告警系统。
- 本轮不执行真实生产环境发布。
- 本轮不引入外部队列或工作流编排系统。
- 本轮不开放任意写操作 capability。

## 4. 验收标准

- Web 或 service 层可以获取结构化部署演练结果。
- 成本与运行趋势具备持久化或可稳定复用的聚合来源。
- Agent 告警策略可配置，并能在审计记录中体现决策结果。
- 企业微信通知具备组件化动作结构或等价 fallback。
- `go test ./...`、`go vet ./...`、`npm --prefix web run test`、`npm --prefix web run type-check`、`npm --prefix web run build` 通过。

## 5. 本轮实施清单

1. 后端任务工作台响应补充部署演练结果、长期趋势快照、告警策略摘要和企业微信动作组件 fallback 结构。
2. 告警摘要按可配置策略进行过滤和决策，并将策略命中结果写入任务工作台 audit。
3. 长期趋势以稳定快照结构输出，覆盖成本、失败、通知、恢复和接管指标。
4. 企业微信启动、过程、审批和完成通知输出可解析的动作组件 fallback，覆盖查看进度、审批、重试、恢复和取消语义。
5. Web API 类型与任务工作台展示新增结构，支持用户查看演练、趋势、策略和动作组件摘要。
6. 补充后端测试覆盖新增结构和告警策略决策，并执行完整验证。

## 6. 实施结果

1. 后端任务工作台响应已补充 `alert_policy`、`trend_snapshot`、`drill`、`wechat_components`，并继续保留既有 `alerts`、`cost_trend`、`deployment`。
2. 告警策略基于用户通知偏好生成启用与静默决策，并写入 `agent.alert_policy_decision` audit。
3. 生产部署演练快照基于部署验收、长期趋势、告警策略审计和企微动作 fallback 生成，并写入 `agent.production_drill_snapshot` audit。
4. 长期趋势快照基于持久化 plan metadata、scheduled task 和 audit log 输出，覆盖成本、失败、通知失败、恢复和接管指标。
5. 企业微信启动、过程、审批和完成通知已输出稳定 `企微动作组件` fallback，覆盖查看进度、审批、重试、恢复和取消语义。
6. Web API 类型和任务工作台已展示告警策略、长期趋势、生产演练和企微动作组件摘要。
