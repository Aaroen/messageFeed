# Agent 生产发布审批按钮回调写审计日报发送与监控告警计划

**创建日期**：2026-06-25

## 1. 本轮目标

在 Agent 已具备任务入口、实时进度、企业微信原生发送 payload、按钮闭环摘要、写能力受控执行摘要、生产日报持久化摘要和上线后监控摘要的基础上，推进生产发布审批执行、真实按钮回调处理、写能力审计复盘、日报定时发送和监控告警实测，使 Agent 从预发布可观察状态进一步进入可审批上线、可交互闭环和可运营告警状态。

## 2. 实施范围

1. 生产发布审批执行：
   - 将上线审批从摘要推进到可执行审批流。
   - 覆盖审批请求、审批结果、过期处理、拒绝路径、回滚路径和审计记录。
2. 真实按钮回调处理：
   - 接入企业微信按钮回调动作处理。
   - 覆盖查看进度、审批、重试、恢复、取消和最终报告。
   - 保留文本 fallback 路径。
3. 写能力审计复盘：
   - 对 `agent.schedule_message` 与 `agent.schedule_task` 的写操作形成审计复盘摘要。
   - 覆盖审批、预算、权限、执行、失败、回滚和人工接管。
4. 日报定时发送：
   - 将生产日报持久化记录推进到定时发送。
   - 覆盖生成、持久化、投递、失败重试和企微汇报。
5. 监控告警实测：
   - 对上线后监控摘要推进到告警触发实测。
   - 覆盖健康、SLA、错误、成本、队列、worker、通知失败和人工接管。

## 3. 非目标

- 本轮不开放任意未列入白名单的写 capability。
- 本轮不移除企业微信文本 fallback。
- 本轮不引入外部监控平台作为强依赖。
- 本轮不重构既有 Agent 计划、运行和审批领域模型。

## 4. 验收标准

- 生产发布审批具有可执行结果和审计记录。
- 企业微信按钮回调可以映射到真实任务动作，并保留 fallback。
- 写能力审计复盘可回溯审批、预算、权限、执行和回滚证据。
- 生产日报可以通过定时任务发送，并记录发送结果。
- 监控告警实测可在 Web 或 service 层查看触发依据与通知结果。
- `go test ./...`、`go vet ./...`、`npm --prefix web run test`、`npm --prefix web run type-check`、`npm --prefix web run build` 通过。

## 5. 后续衔接

本轮完成后，继续推进真实生产发布窗口、企业微信端到端人工验收、写能力灰度扩容和外部监控平台对接。

2. 生产发布审批执行摘要已覆盖审批请求、审批结果、过期处理、拒绝路径、回滚路径、可执行状态和审计事件。
