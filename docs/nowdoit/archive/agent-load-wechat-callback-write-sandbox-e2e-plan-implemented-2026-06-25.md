# Agent 压测企微回调写操作沙箱与端到端验收计划

**创建日期**：2026-06-25

## 1. 本轮目标

在 Agent 已具备 Web/企业微信任务入口、实时进度、任务工作台、部署演练快照、长期趋势、告警策略决策和企业微信动作 fallback 的基础上，推进多用户生产压测、真实企业微信回调联调准备、写操作 capability 审批沙箱和端到端验收脚本，使 Agent 从上线准备状态进一步进入可压测、可联调、可受控扩权和可重复验收阶段。

## 2. 实施范围

1. 多用户生产压测：
   - 增加多用户任务入口、准入、配额、执行和进度聚合的压测摘要结构。
   - 覆盖 Web、企业微信、定时任务和恢复路径。
   - 保留可在 Web 任务工作台查看的压测结果。
2. 企业微信真实回调联调：
   - 梳理并结构化输出回调验收项，包括签名、解密、幂等、绑定、回复、异步通知和最终汇报。
   - 保留文本 fallback 与动作组件 fallback 的一致性检查。
3. 写操作 capability 审批沙箱：
   - 为未来写操作 capability 增加沙箱策略结构。
   - 默认保持拒绝或需审批，不开放任意写操作。
   - 审批、预算、权限和审计信息需进入计划 metadata 或 audit。
4. 端到端验收脚本化：
   - 建立可由 service 或测试触发的端到端验收摘要。
   - 覆盖 Web 发起任务、企微发起任务、实时进度、审批、恢复、完成汇报和审计回溯。

## 3. 非目标

- 本轮不执行真实生产压测流量。
- 本轮不直接接入生产企业微信回调地址。
- 本轮不开放无审批写操作 capability。
- 本轮不引入外部压测平台。

## 4. 验收标准

- Web 或 service 层可以查看多用户压测摘要。
- 企业微信回调联调项具备结构化验收输出。
- 写操作 capability 默认受沙箱策略约束，并可在 metadata 或 audit 中回溯。
- 端到端验收摘要覆盖任务发起、进度、审批、恢复、汇报和审计链路。
- `go test ./...`、`go vet ./...`、`npm --prefix web run test`、`npm --prefix web run type-check`、`npm --prefix web run build` 通过。

## 5. 本轮实施清单

1. 后端任务工作台响应补充多用户压测摘要、企业微信回调联调摘要、写操作 capability 沙箱摘要和端到端验收摘要。
2. 多用户压测摘要覆盖 Web、企业微信、定时任务、恢复、准入受限和配额受限指标。
3. 企业微信回调联调摘要覆盖签名、解密、幂等、绑定、回复、异步通知、最终汇报和动作 fallback。
4. 写操作 capability 沙箱默认拒绝或需审批，并将策略快照写入 audit。
5. 端到端验收摘要覆盖 Web 发起、企微发起、实时进度、审批、恢复、完成汇报和审计回溯。
6. Web API 类型和任务工作台展示新增摘要，补充测试并执行完整验证。

## 6. 实施结果

1. 后端任务工作台响应已补充 `load_test`、`wechat_callback`、`write_sandbox`、`e2e` 四类结构化摘要。
2. 多用户压测摘要基于 plan、scheduled task 和 audit 聚合用户数、Web 任务、企微任务、定时任务、恢复、准入受限、配额受限和进度事件指标。
3. 企业微信回调联调摘要已覆盖签名、解密、幂等、绑定、回复、异步通知、最终汇报和动作 fallback 检查项。
4. 写操作 capability 沙箱默认策略为 `reject_or_require_approval`，并输出默认拒绝、审批、预算、权限审计和沙箱范围检查项。
5. 端到端验收摘要覆盖 Web 发起、企微发起、实时进度、审批、恢复、定时任务、最终汇报、审计回溯和企微动作 fallback。
6. 写操作沙箱快照写入 `agent.write_sandbox_snapshot` audit，端到端验收快照写入 `agent.e2e_acceptance_snapshot` audit。
7. Web API 类型和任务工作台已展示多用户压测、企微回调联调、写操作沙箱和端到端验收摘要。
