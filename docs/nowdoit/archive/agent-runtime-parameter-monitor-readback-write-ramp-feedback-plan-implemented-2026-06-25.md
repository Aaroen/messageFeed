# Agent 运行参数监控回读写放量反馈计划

**创建日期**：2026-06-25

## 1. 本轮目标

在 Agent 已具备 Web/企业微信任务入口、实时进度、企业微信按钮直接控制、生产真实执行记录、外部监控平台联调、写能力放量策略参数、企业微信最终汇报和上线运行总览摘要的基础上，推进上线运行参数固化、真实监控数据回读、写能力逐步放量和企业微信用户侧持续运营反馈，使 Agent 上线运行状态具备更稳定的参数、监控证据和用户反馈闭环。

## 2. 实施范围

1. 运行参数固化：
   - 将上线运行总览推进为可查询的运行参数摘要。
   - 覆盖放量比例、用户范围、通知通道、监控通道、审批门禁和回滚阈值。
2. 真实监控数据回读：
   - 将外部监控联调状态推进为监控回读摘要。
   - 覆盖指标回读、事件回读、告警状态、日报状态和数据新鲜度。
3. 写能力逐步放量：
   - 对 `agent.schedule_message` 与 `agent.schedule_task` 建立下一阶段放量建议摘要。
   - 覆盖当前比例、建议比例、限制条件、回滚条件和默认拒绝策略。
4. 企业微信用户侧持续反馈：
   - 将最终汇报推进为用户反馈摘要。
   - 覆盖完成反馈、失败反馈、按钮反馈、Web 追踪反馈和下一步处理。
5. 运行运营闭环：
   - 汇总运行参数、监控回读、写放量建议和用户反馈状态。

## 3. 非目标

- 本轮不开放未进入白名单的写 capability。
- 本轮不移除企业微信文本 fallback。
- 本轮不执行不可回滚的真实生产变更。
- 本轮不绑定单一外部监控 SaaS 的私有 API。

## 4. 验收标准

- 运行参数摘要可在 Web 或 service 层查看，并写入审计事件。
- 监控回读摘要可查询指标、事件、告警、日报和数据新鲜度。
- 写能力放量建议继续保持最小权限、审批、预算、审计和回滚约束。
- 企业微信用户反馈摘要覆盖完成、失败、按钮、Web 追踪和下一步处理。
- 运行运营闭环纳入运行参数、监控回读、写放量建议和用户反馈状态。
- `go test ./...`、`go vet ./...`、`npm --prefix web run test`、`npm --prefix web run type-check`、`npm --prefix web run build` 通过。

## 5. 后续衔接

本轮完成后，继续推进可配置运营面板、监控异常自动汇报、写能力分阶段放量和企业微信用户侧闭环运营。

## 6. 实施结果

1. 后端任务工作台已新增五类运行运营摘要：
   - `runtime_parameters`：运行参数摘要，覆盖放量比例、用户范围、通知通道、监控通道、审批门禁、预算门禁和回滚阈值。
   - `monitor_readback`：监控回读摘要，覆盖指标回读、事件回读、告警状态、日报状态和数据新鲜度。
   - `write_ramp_recommendation`：写放量建议摘要，覆盖当前比例、建议比例、限制条件、回滚条件和默认拒绝策略。
   - `wechat_user_feedback`：企微用户反馈摘要，覆盖完成反馈、失败反馈、按钮反馈、Web 追踪反馈和下一步处理。
   - `operations_runtime_closure`：运行运营闭环摘要，覆盖运行参数、监控回读、写放量建议、企微用户反馈和审计状态。
2. 后端审计已新增五类快照事件：
   - `agent.runtime_parameters_snapshot`
   - `agent.monitor_readback_snapshot`
   - `agent.write_ramp_recommendation_snapshot`
   - `agent.wechat_user_feedback_snapshot`
   - `agent.operations_runtime_closure_snapshot`
3. 前端任务工作台已新增运行运营展示：
   - API 类型定义已同步新增五类响应结构。
   - `AgentPlanView.vue` 已展示运行参数、监控回读、写放量建议、企微用户反馈和运行运营闭环摘要及检查项。
4. 服务层测试已覆盖新增响应字段和审计事件。
