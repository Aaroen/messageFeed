# Agent 部署验收告警成本趋势与企微交互计划

**创建日期**：2026-06-25

## 1. 本轮目标

在 Agent 已具备 Web/企微任务入口、实时进度、恢复治理、质量评分、用户策略、capability 策略、SLA、成本摘要、配额和多节点一致性 metadata 的基础上，推进生产部署脚本化验收、运行告警、成本趋势留存和更细粒度的企业微信交互式进度能力，使 Agent 在上线前后具备可验证、可告警、可追踪和可交互的运行闭环。

## 2. 实施范围

1. 部署脚本化验收：
   - 将现有部署验收 metadata 扩展为可由后端接口或 service helper 输出的结构化验收结果。
   - 覆盖迁移、healthz、worker、Web 入口、企微入口、限流、配额、通知和 eval。
   - 保持现有 Docker 与数据库配置不做破坏性调整。
2. 运行告警：
   - 为任务失败、配额超限、通知失败、低质量评分和接管待处理建立告警摘要。
   - Web 任务工作台展示当前告警数量和告警原因。
   - 企微最终汇报或过程通知可包含关键告警状态。
3. 成本趋势留存：
   - 基于现有 plan metadata 和 audit，生成近期成本趋势摘要。
   - 覆盖工具调用、外部访问、估算 token、通知次数和重试次数。
   - Web 展示近期趋势摘要。
4. 企业微信交互式进度：
   - 在企微文本通知中增加更稳定的操作提示和状态锚点。
   - 为审批、恢复、取消、重试和查看详情保留统一进度 URL 与动作文本。

## 3. 非目标

- 本轮不引入外部告警系统。
- 本轮不执行真实生产部署。
- 本轮不引入外部队列系统。
- 本轮不开放任意写操作 capability。

## 4. 验收标准

- 部署验收可通过结构化 service/helper 输出，并在 progress 或 Web 中可见。
- Web 任务工作台能展示告警摘要和成本趋势摘要。
- 企微通知包含明确状态锚点、进度地址和下一步动作。
- 成本趋势基于现有任务 metadata 或 audit 可计算。
- `go test ./...`、`go vet ./...`、`npm --prefix web run test`、`npm --prefix web run type-check`、`npm --prefix web run build` 通过。

## 5. 本轮实施清单

1. 后端任务列表响应补充结构化运行告警、近 7 日成本趋势和部署验收字段。
2. Web API 类型补充 `alerts`、`cost_trend`、`deployment` 响应结构。
3. Web 任务工作台展示告警摘要、成本趋势摘要和部署验收摘要。
4. 企业微信进度、启动、审批和完成文本统一输出状态锚点、进度地址和下一步动作。
5. 后端测试覆盖任务列表新增字段和企业微信通知关键文本。
6. 完整执行 Go、vet、Web 单元测试、类型检查和构建验证。

## 6. 实施结果

1. 后端任务列表响应已输出 `alerts`、`cost_trend`、`deployment`，并基于 plan metadata、scheduled task 和 audit log 计算运行告警、近 7 日成本趋势与部署验收摘要。
2. Web API 类型已补充运行告警、成本趋势和部署验收结构，任务工作台已展示对应摘要和部署检查项。
3. 企业微信启动、过程、审批和完成文本已统一包含状态锚点、进度地址和下一步动作提示。
4. 后端测试已覆盖任务列表新增字段，以及企业微信通知文本中的状态锚点、进度地址和下一步动作。
