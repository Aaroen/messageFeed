# Agent 企业微信进度卡片 Web 证据交互回放权限与恢复审计计划

**创建日期**：2026-06-25

## 1. 本轮目标

在 Agent 已具备 Web/企业微信任务入口、实时进度、企业微信按钮控制、运营证据闭环摘要以及双端进度证据摘要的基础上，推进企业微信可视化进度卡片细化、Web 证据明细交互、回调重放权限控制和恢复策略变更审计，使双端用户看到的进度、证据、回放和恢复策略具备更明确的交互状态和权限边界。

## 2. 实施范围

1. 企业微信可视化进度卡片摘要：
   - 将统一进度组件推进为企业微信卡片摘要。
   - 覆盖卡片键、阶段状态、进度百分比、详情入口、操作按钮和 fallback 文本。
2. Web 证据明细交互摘要：
   - 将证据明细页推进为 Web 交互摘要。
   - 覆盖筛选条件、明细展开、回放入口、审计展示、保留策略提示和用户可见性。
3. 回调重放权限控制摘要：
   - 将回调重放工具推进为权限控制摘要。
   - 覆盖权限键、允许角色、幂等保护、验签复核、失败降级和审计事件。
4. 恢复策略变更审计摘要：
   - 将恢复策略配置推进为变更审计摘要。
   - 覆盖变更键、旧策略、新策略、审批状态、回滚路径和审计证据。
5. 双端交互治理总览：
   - 汇总企业微信进度卡片、Web 证据交互、回调重放权限和恢复策略变更审计状态。

## 3. 非目标

- 本轮不开放未进入白名单的写 capability。
- 本轮不绕过审批、预算、权限、审计和回滚约束。
- 本轮不执行不可回滚的真实生产变更。
- 本轮不移除企业微信文本 fallback。

## 4. 验收标准

- 企业微信可视化进度卡片摘要可在 service 层和 Web 任务工作台查看，并写入审计事件。

- 回调重放权限控制摘要覆盖权限、角色、幂等、验签、失败降级和审计事件。

- 双端交互治理总览纳入企业微信进度卡片、Web 证据交互、回调重放权限和恢复策略变更审计状态。
