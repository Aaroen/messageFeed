# Agent 运营 API 执行异常回执写审批按钮与工单 SLA 计划

**创建日期**：2026-06-25

## 1. 本轮目标

在 Agent 已具备 Web/企业微信任务入口、实时进度、企业微信按钮控制、运营面板配置、监控异常自动汇报、写能力分阶段放量、企业微信反馈处理、运营闭环总览、运营处理摘要以及运营动作闭环摘要的基础上，推进运营动作真实 API 执行、异常升级通知投递回执、写阶段审批按钮化和反馈工单处理 SLA，使运营动作从可审计定义推进到可执行、可确认、可追踪的闭环。

## 2. 实施范围

1. 运营动作 API 执行摘要：
   - 将运营动作定义推进为 API 执行摘要。
   - 覆盖动作键、执行入口、执行状态、权限检查、幂等结果和审计事件。
2. 异常升级通知投递回执：
   - 将异常升级策略推进为通知回执摘要。
   - 覆盖通知通道、接收方、投递状态、重复抑制结果、恢复通知状态和失败转人工入口。
3. 写阶段审批按钮化：
   - 将写阶段审批推进为企业微信与 Web 可确认按钮摘要。
   - 覆盖按钮键、审批状态、权限范围、回滚门槛、拒绝路径和审计证据。
4. 反馈工单处理 SLA：
   - 将反馈工单生命周期推进为 SLA 摘要。
   - 覆盖首响时限、处理时限、超时升级、用户等待、关闭条件和失败转人工路径。
5. 运营执行闭环总览：
   - 汇总运营动作 API 执行、异常通知回执、写审批按钮和反馈工单 SLA 状态。

## 3. 非目标

- 本轮不开放未进入白名单的写 capability。
- 本轮不绕过审批、预算、权限、审计和回滚约束。
- 本轮不执行不可回滚的真实生产变更。
- 本轮不移除企业微信文本 fallback。

## 4. 验收标准

- 运营动作 API 执行摘要可在 service 层和 Web 任务工作台查看，并写入审计事件。
- 异常升级通知投递回执覆盖通道、接收方、投递状态、抑制结果、恢复通知和失败转人工入口。
- 写阶段审批按钮摘要覆盖企业微信与 Web 确认入口，并保留默认拒绝策略。
- 反馈工单 SLA 摘要覆盖时限、超时升级、等待用户、关闭条件和失败转人工路径。
- 运营执行闭环总览纳入 API 执行、通知回执、写审批按钮和工单 SLA 状态。
- `go test ./...`、`go vet ./...`、`npm --prefix web run test`、`npm --prefix web run type-check`、`npm --prefix web run build` 通过。
