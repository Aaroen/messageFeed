# 阶段二 Web Feed 收尾执行计划

**最后更新**：2026-06-25

## 目标

完成 `docs/implementation.md` 中 P0 阶段二收尾事项，使 Web 阅读闭环从“可查询”推进到“可操作、可筛选、可分页、可持久化偏好”：

- Web 界面支持条目已读、收藏、隐藏和取消状态。
- Web 界面支持时间线筛选、分页加载和阅读模式偏好持久化。
- 提供独立 `/items/:id` 详情路由。
- 抽取可复用 `ActionBar` 状态操作组件。
- 对齐现有 OpenAPI 契约和后端接口行为。

## P0 实施范围

1. 核实后端条目状态接口：
   - `POST /api/v1/items/{id}/read`
   - `POST /api/v1/items/{id}/favorite`
   - `POST /api/v1/items/{id}/hide`
   - `GET /api/v1/items`
   - `GET /api/v1/items/{id}`
   - `GET /api/v1/feed/timeline`
   - `GET /api/v1/feed/view-mode`
   - `PUT /api/v1/feed/view-mode`
2. 补齐前端 API 调用封装，避免组件中散落请求细节。
3. 在时间线列表中加入筛选参数：
   - 来源过滤。
   - 已读过滤。
   - 收藏过滤。
   - 隐藏过滤。
   - limit / offset 分页。
4. 实现条目状态操作：
   - 标记已读与取消已读。
   - 收藏与取消收藏。
   - 隐藏与取消隐藏。
   - 操作后局部更新列表状态。
5. 实现 `/items/:id` 详情页：
   - 展示标题、来源、发布时间、摘要或正文片段。
   - 提供阅读原文入口。
   - 复用状态操作组件。
6. 持久化 Web 阅读模式偏好：
   - 初始化时读取后端偏好。
   - 切换时间线/推荐模式时写回后端。
   - 请求失败时保留可解释错误状态。

## 非目标

- 不新增删除订阅源功能。
- 不实现推荐算法重构。
- 不实现离线全文抽取。
- 不新增复杂 E2E 框架；仅补充必要的现有测试或类型检查。
- 不引入新的前端 UI 框架。

## 验收标准

- 时间线页面可以按来源、已读、收藏、隐藏状态筛选。
- 分页加载不会重复展示同一条目，空结果有稳定状态。
- 条目卡片和详情页均可执行已读、收藏、隐藏及对应取消操作。
- 状态操作调用后，前端显示状态与后端返回结果一致。
- `/items/:id` 可直接访问并在无记录时显示错误状态。
- 阅读模式偏好刷新页面后仍保持最近选择。
- `api/openapi.yaml` 与实际使用接口保持一致。
- `go test ./...` 通过。
- `npm --prefix web run type-check` 通过。

## 实施顺序

1. 核实后端接口与 OpenAPI 契约，必要时只做兼容性修补。
2. 梳理 Web 当前页面、路由、状态管理与 API 封装。
3. 补齐阅读模式偏好读取和写回。
4. 实现筛选与分页参数联动。
5. 抽取并接入 `ActionBar`。
6. 实现条目详情路由与详情页状态操作。
7. 运行后端测试、前端类型检查和必要的构建验证。

## 后续衔接

本计划完成后，下一 P0 包应进入阶段三收尾：完整 Compose 环境端到端观测验收，重点验证 `request_id`、`trace_id`、日志、指标和 trace 的串联关系。
