# Agent 企微原生联调写操作回放上线审批与生产日报计划

**创建日期**：2026-06-25

## 1. 本轮目标

在 Agent 已具备 Web/企业微信任务入口、实时进度、任务工作台、企业微信原生按钮 schema、原生发送 payload、写能力灰度策略、生产级告警通道和上线演练记录的基础上，推进真实企业微信按钮消息联调、灰度写操作执行回放、上线审批流和生产运行日报，使 Agent 更接近可审计、可批准、可回放和可日常运营的生产状态。

## 2. 实施范围

1. 真实企业微信按钮联调：
   - 输出企业微信按钮 payload 联调摘要。
   - 覆盖 payload 生成、文本 fallback、动作 URL、审批、重试、恢复、取消和最终报告。
   - 记录联调风险、阻断项和下一步动作。
2. 灰度写操作执行回放：
   - 为 `agent.schedule_message` 与 `agent.schedule_task` 输出执行回放摘要。
   - 覆盖审批、预算、权限、执行结果、审计记录和回滚条件。
   - 默认不开放其他写操作。
3. 上线审批流：
   - 输出上线审批摘要，覆盖申请、评审、批准、拒绝、过期和审计。
   - 保留人工接管与回滚路径。
4. 生产运行日报：
   - 输出生产运行日报摘要，覆盖任务量、成功率、失败、告警、成本、趋势、接管、恢复和通知。
   - 支持 Web 任务工作台查看。

## 3. 非目标

- 本轮不执行真实生产发布。
- 本轮不开放任意写操作 capability。
- 本轮不移除文本 fallback。
- 本轮不接入外部日报系统。

## 4. 验收标准

- Web 或 service 层可以查看企业微信按钮联调摘要。
- 灰度写操作回放摘要覆盖审批、预算、权限、执行、审计和回滚。
- 上线审批摘要覆盖申请、评审、批准、拒绝、过期和审计。
- 生产运行日报摘要覆盖任务量、成功率、失败、告警、成本、趋势、接管、恢复和通知。
- `go test ./...`、`go vet ./...`、`npm --prefix web run test`、`npm --prefix web run type-check`、`npm --prefix web run build` 通过。

## 5. 本轮实施清单

1. 后端任务工作台响应补充企业微信按钮联调摘要、灰度写操作执行回放摘要、上线审批摘要和生产运行日报摘要。
2. 企业微信按钮联调摘要覆盖 payload 生成、文本 fallback、动作 URL、审批、重试、恢复、取消和最终报告，并输出风险、阻断项和下一步动作。
3. 灰度写操作回放摘要覆盖 `agent.schedule_message` 与 `agent.schedule_task` 的审批、预算、权限、执行结果、审计记录和回滚条件。
4. 上线审批摘要覆盖申请、评审、批准、拒绝、过期、人工接管、回滚路径和审计。
5. 生产运行日报摘要覆盖任务量、成功率、失败、告警、成本、趋势、接管、恢复和通知。
6. Web API 类型和任务工作台展示新增摘要，补充测试并执行完整验证。
