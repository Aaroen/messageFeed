# Agent 统一进度证据明细回调重放与恢复配置计划

**创建日期**：2026-06-25

## 1. 本轮目标

在 Agent 已具备 Web/企业微信任务入口、实时进度、企业微信按钮控制、运营面板配置、运营执行闭环摘要以及运营证据闭环摘要的基础上，推进 Web/企业微信统一进度组件、执行证据明细页、回调重放工具和运营恢复策略配置，使用户在 Web 和企业微信两端看到的进度、证据和恢复入口保持一致。

## 2. 实施范围

1. Web/企业微信统一进度组件摘要：
   - 将现有实时进度、任务摘要和企业微信状态推进为统一组件摘要。
   - 覆盖组件键、Web 展示状态、企业微信展示状态、事件游标、刷新策略和审计证据。
2. 执行证据明细页摘要：
   - 将运营证据闭环推进为明细页摘要。
   - 覆盖明细入口、记录数量、审计事件、回放入口、用户可见性和数据保留策略。
3. 回调重放工具摘要：
   - 将企业微信审批回调入库推进为回调重放摘要。
   - 覆盖回调键、重放入口、验签复核、幂等保护、失败降级和审计证据。
4. 运营恢复策略配置摘要：
   - 将异常自动恢复推进为策略配置摘要。
   - 覆盖策略键、恢复触发、抑制窗口、重开条件、人工接管和默认保守策略。
5. 双端进度证据总览：
   - 汇总统一进度组件、证据明细页、回调重放工具和恢复策略配置状态。

## 3. 非目标

- 本轮不开放未进入白名单的写 capability。
- 本轮不绕过审批、预算、权限、审计和回滚约束。
- 本轮不执行不可回滚的真实生产变更。
- 本轮不移除企业微信文本 fallback。

## 4. 验收标准
