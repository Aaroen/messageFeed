# Agent 运营执行持久化审批回调 SLA 报表与自动恢复计划

**创建日期**：2026-06-25

## 1. 本轮目标

在 Agent 已具备 Web/企业微信任务入口、实时进度、企业微信按钮控制、运营面板配置、监控异常自动汇报、写能力分阶段放量、企业微信反馈处理、运营闭环总览、运营动作闭环摘要以及运营执行闭环摘要的基础上，推进运营动作持久化执行记录、企业微信审批回调入库、工单 SLA 统计报表和异常升级自动恢复闭环，使运营执行从可显示摘要推进到可追溯、可回放、可统计的运行证据。

## 2. 实施范围

1. 运营动作持久化执行记录：
   - 将运营动作 API 执行摘要推进为持久化执行记录摘要。
   - 覆盖记录键、动作键、执行状态、幂等状态、审计事件和回放入口。
2. 企业微信审批回调入库：
   - 将写审批按钮摘要推进为回调入库摘要。
   - 覆盖回调键、来源通道、审批结果、验签状态、入库状态和失败降级路径。
3. 工单 SLA 统计报表：
   - 将反馈工单 SLA 摘要推进为统计报表摘要。
   - 覆盖首响达标率、处理达标率、超时数量、等待用户数量、转人工数量和报表审计事件。
4. 异常升级自动恢复闭环：
   - 将异常升级回执推进为自动恢复摘要。
   - 覆盖恢复触发、恢复通知、抑制解除、重复告警重开、人工接管和审计证据。
5. 运营证据闭环总览：
   - 汇总持久化执行记录、审批回调入库、SLA 报表和自动恢复闭环状态。

## 2.1 本轮实施清单

## 3. 非目标

- 本轮不开放未进入白名单的写 capability。
- 本轮不绕过审批、预算、权限、审计和回滚约束。
- 本轮不执行不可回滚的真实生产变更。
- 本轮不移除企业微信文本 fallback。

## 4. 验收标准

- 运营动作持久化执行记录摘要可在 service 层和 Web 任务工作台查看，并写入审计事件。
- 企业微信审批回调入库摘要覆盖回调来源、验签、入库和失败降级路径。
- 工单 SLA 统计报表摘要覆盖达标率、超时、等待用户和转人工计数。
- 异常升级自动恢复闭环摘要覆盖恢复通知、抑制解除、重开和人工接管路径。
- 运营证据闭环总览纳入持久化执行、审批回调、SLA 报表和自动恢复状态。
- `go test ./...`、`go vet ./...`、`npm --prefix web run test`、`npm --prefix web run type-check`、`npm --prefix web run build` 通过。

## 6. 实施结果

1. 后端任务工作台新增五类运营证据闭环摘要：
   - `ops_execution_record`：覆盖记录键、动作键、执行状态、幂等状态、审计事件和回放入口。
   - `wechat_approval_callback`：覆盖回调键、来源通道、审批结果、验签状态、入库状态和失败降级路径。

   - `alert_auto_recovery`：覆盖恢复触发、恢复通知、抑制解除、重复告警重开、人工接管和审计证据。
   - `operations_evidence`：汇总持久化执行记录、企业微信审批回调、SLA 报表、异常自动恢复和审计状态。
