# Agent 真实交互自动化执行计划

**创建日期**：2026-06-25

## 1. 本轮目标

在双端闭环摘要和第一阶段结构治理完成后，推进企业微信模板试运行指标落库、Web 证据页面实际操作 API、回放执行结果查询 API 和恢复策略自动化执行，使用户从企业微信或 Web 发起任务后，双端进度、证据、回放和恢复链路从摘要展示进一步进入可操作状态。

## 2. 实施范围

1. 企业微信模板试运行指标：
   - 记录模板试运行批次、目标用户、发送状态、fallback 命中、消息 ID 和审计引用。
   - 在任务聚合结果中提供可查询的试运行指标摘要。
2. Web 证据页面实际操作 API：
   - 支持证据筛选、展开、时间线查看和回放申请操作的服务层入口。
   - 将用户操作写入审计记录，供 Web 任务工作台展示。
3. 回放执行结果查询：
   - 提供回放请求、执行结果、幂等命中、审批决策、签名校验和失败原因查询。
   - 保持回放执行受权限、审批和审计约束。
4. 恢复策略自动化执行：
   - 根据灰度比例、暂停条件、回滚条件和审计状态生成自动推进建议。
   - 保持默认保守策略，不绕过人工审批和回滚约束。
5. 双端可视化衔接：
   - Web 任务工作台显示真实操作记录。
   - 企业微信后续汇报可引用试运行指标、证据操作、回放结果和恢复策略状态。

## 2.1 本轮实施清单

1. 梳理现有 repository/service/handler 中可复用的审计与任务查询入口。
2. 新增企业微信模板试运行指标服务模型和聚合摘要。
3. 新增 Web 证据操作服务入口和审计事件。
4. 新增回放执行结果查询服务入口。
5. 新增恢复策略自动化执行建议服务入口。
6. 同步前端 API 类型与任务工作台展示。
7. 完成验证矩阵：
   - `go test ./...`
   - `go vet ./...`
   - `npm --prefix web run test`
   - `npm --prefix web run type-check`
   - `npm --prefix web run build`
8. 记录实施结果和验证记录，归档本文档并创建下一轮文档。

## 3. 非目标

- 本轮不直接执行不可回滚的真实生产写操作。
- 本轮不绕过认证、权限、审批、预算、审计和回滚约束。
- 本轮不移除企业微信文本 fallback。
- 本轮不扩大未纳入策略白名单的写 capability。

## 4. 验收标准

- 企业微信模板试运行指标可由后端服务生成并在 Web 任务工作台查看。
- Web 证据操作具备服务层入口和审计记录。
- 回放执行结果可查询，并包含幂等、审批、签名和失败原因字段。
- 恢复策略自动化执行建议可查询，并保留暂停与回滚约束。
- 完整验证矩阵通过。

## 5. 后续衔接

本轮完成后，继续推进企业微信真实模板发送适配、Web 证据操作端到端交互、回放执行审批闭环和恢复策略自动推进落库。
