# Agent 企业微信模板试运行证据操作回放结果与恢复自动化计划

**创建日期**：2026-06-25

## 1. 本轮目标

在 Agent 已具备企业微信模板联调状态、Web 证据交互细节、回放执行安全审计和恢复策略灰度发布摘要的基础上，推进真实环境企业微信模板发送试运行、证据页面用户操作、回放执行结果留痕和恢复策略灰度自动化，使用户从企业微信或 Web 发起任务后，双端进度、细节、回放和恢复链路进一步接近可运行闭环。

## 2. 实施范围

1. 企业微信模板试运行：
   - 汇总试运行批次、目标用户范围、模板发送状态、fallback 命中、消息 ID 和审计证据。
2. Web 证据页面用户操作：
   - 汇总筛选操作、展开操作、审计时间线查看、回放申请操作和权限校验结果。
3. 回放执行结果留痕：
   - 汇总执行结果、幂等命中、审批决策、签名校验、失败原因和审计记录。
4. 恢复策略灰度自动化：
   - 汇总自动推进条件、暂停条件、回滚条件、当前比例、下一比例和审计证据。
5. 双端任务闭环总览：
   - 汇总企业微信试运行、Web 证据操作、回放结果留痕和恢复灰度自动化状态。

## 2.1 本轮实施清单

1. 后端任务聚合结果新增五类闭环推进摘要：
   - 企业微信模板试运行摘要。
   - Web 证据页面用户操作摘要。
   - 回放执行结果留痕摘要。
   - 恢复策略灰度自动化摘要。
   - 双端任务闭环总览。
2. 后端 builder 基于上一轮运行闭环摘要继续派生本轮摘要：
   - 基于 `wechat_template_integration` 派生试运行批次、目标用户范围、模板发送状态、fallback 命中、消息 ID 和审计证据。
   - 基于 `web_evidence_interaction_detail` 派生筛选操作、展开操作、审计时间线查看、回放申请操作和权限校验结果。
   - 基于 `callback_replay_safety_audit` 派生执行结果、幂等命中、审批决策、签名校验、失败原因和审计记录。
   - 基于 `recovery_policy_gray_release` 派生自动推进条件、暂停条件、回滚条件、当前比例、下一比例和审计证据。
   - 汇总企业微信试运行、Web 证据操作、回放结果留痕和恢复灰度自动化状态。
3. `ListTasks` 接入本轮五类摘要，并写入对应审计快照事件。
4. 服务层测试补充本轮新增字段和审计事件断言。
5. 前端 API 类型和 Agent 任务工作台展示本轮新增摘要。
6. 完成完整验证矩阵：
   - `go test ./...`
   - `go vet ./...`
   - `npm --prefix web run test`
   - `npm --prefix web run type-check`
   - `npm --prefix web run build`
7. 将实施结果和验证记录追加到本文档，随后归档本文档并创建下一轮唯一活动文档。

## 3. 非目标

- 本轮不移除企业微信文本 fallback。
- 本轮不绕过认证、审批、权限、预算、审计和回滚约束。
- 本轮不执行不可回滚的真实生产操作。
- 本轮不开放未纳入策略白名单的写 capability。

## 4. 验收标准

- 企业微信模板试运行状态可在 service 层和 Web 任务工作台查看。
- Web 证据用户操作覆盖筛选、展开、时间线、回放申请和权限校验。
- 回放执行结果留痕覆盖执行结果、幂等、审批、签名、失败原因和审计。
- 恢复策略灰度自动化覆盖推进、暂停、回滚、当前比例、下一比例和审计。
- 双端任务闭环总览纳入企业微信试运行、Web 证据操作、回放结果和恢复自动化状态。
- `go test ./...`、`go vet ./...`、`npm --prefix web run test`、`npm --prefix web run type-check`、`npm --prefix web run build` 通过。
